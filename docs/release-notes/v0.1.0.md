# AzImg CLI v0.1.0

Initial public preview of `azimg`.
It targets agents, scripts, CI, and unattended shells.

## Highlights

- Generate images from text prompts with Azure OpenAI.
- Edit existing images with optional PNG masks.
- Authenticate through Azure CLI via `AzureCliCredential`.
- Configure reusable profiles for endpoint, deployment, and output paths.
- Override profile values per command for automation-friendly workflows.
- Save images as `png`, `jpeg`, or `webp`.
- Customize count, size, quality, background, compression, and file names.
- Write optional manifest JSON with paths, checksums, usage, and timestamps.
- Emit structured JSON by default, with `--format text` for people.
- Run diagnostics with `azimg doctor` and optional real auth verification.
- Check for and apply updates from GitHub Releases with `azimg update`.
- Install the AzImg AI agent skill with `azimg install-skill`.
- The skill helps AI agents use `azimg` for generation and editing.

## Commands included

- `azimg generate <PROMPT>` creates one or more images from a prompt.
- `azimg edit <FILE> <PROMPT>` edits an image and can use `--mask-file`.
- `azimg doctor` validates config, output setup, and optional Azure auth.
- `azimg config` initializes, shows, and updates local config profiles.
- `azimg install-skill` installs the AzImg AI agent skill.
- `azimg update` checks for or installs a newer released binary.
- `azimg version` prints product and version information.

## Installation assets

This release publishes self-contained single-file binaries for these RIDs:

- `win-x64`
- `win-arm64`
- `osx-arm64`
- `linux-x64`

Each release also includes `SHA256SUMS` and `azimg-release.json`.
Install scripts and self-update use them to verify downloaded archives.

## Install scripts

macOS and Linux users can install with `install.sh`.
Windows users can install with `install.ps1`.

Both installers detect the current platform, verify checksums, and install `azimg`.
They also write local metadata under `~/.azimg`.

## Requirements

- Azure CLI must be installed and logged in with `az login`.
- The signed-in identity needs the `Cognitive Services OpenAI User` role.
- A config profile must provide an Azure OpenAI endpoint and deployment.
