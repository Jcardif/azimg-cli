# AzImg CLI v0.1.1

Patch release for the initial `0.1.0` preview.

## Fixes

- Replaced the packaged CLI auth path with a custom Azure CLI token credential
  that avoids the macOS ARM64 .NET pipe crash seen after single-file packaging.
- `azimg doctor --verify-auth`, `azimg generate`, and `azimg edit` still use
  the signed-in Azure CLI account, but no longer depend on Azure Identity's
  redirected async Azure CLI process runner.

## Compatibility

- Configuration, commands, output formats, install scripts, and release assets
  remain compatible with `0.1.0`.
