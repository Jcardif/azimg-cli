# AzImg CLI v0.1.2

Patch release for update reliability and generation progress output.

## Fixes

- Fixed self-update for single-file builds by staging replacement of the running
  executable until after `azimg update` exits.
- Hardened JSON error output so a missing runtime assembly during failure
  reporting falls back to a minimal JSON error document instead of aborting.
- Suppressed noisy Azure CLI Python tracebacks during auth verification and
  replaced them with concise network/auth failure messages.

## Improvements

- Generation and edit commands now use a Spectre.Console one-line progress
  display on stderr instead of writing multiple progress lines.
- Non-interactive and redirected shells remain safe: stdout stays reserved for
  command results, and stderr progress never prompts for input.
