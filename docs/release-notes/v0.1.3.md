# AzImg CLI v0.1.3

Patch release for project-local output defaults and easier config setup.

## Changes

- `azimg config init` now accepts optional `--profile`, `--deployment`, and
  `--endpoint` values so a usable config can be created without opening the
  JSON file manually.
- New starter configs no longer include `outputDirectory` in profiles. Profiles
  are now only for Azure connection settings.
- Image output now defaults to `azimg-output` under the directory where `azimg`
  is run when `generate` or `edit` does not receive `--output-directory`.
- Existing legacy `outputDirectory` values in older profiles are ignored for
  output resolution. Pass `--output-directory` per run when a specific folder is
  required.

## Agent skill

- Updated the AzImg agent skill to strongly recommend passing
  `--output-directory` on image generation and edit commands.
- Added guidance to choose project-local output folders such as
  `./azimg-output` or task-specific subfolders.
